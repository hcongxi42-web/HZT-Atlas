## 一、设计系统升级（Design System v2）

### 1.1 色彩系统重构

当前问题：硬编码色值（`#0a0e1a`、`#334155` 等），无系统性，暗色发灰发脏。

**新色彩架构：**

```css
/* 基础层 — 使用 CSS 变量，支持主题切换 */
:root {
  /* 背景层级 — 从深到浅的 4 层 */
  --bg-base: #030712;           /* 最底层 — 纯黑偏蓝 */
  --bg-elevated: #0B0F1A;       /* 卡片/面板层 */
  --bg-surface: #111827;        /* 交互元素层 */
  --bg-overlay: #1F2937;        /* Hover/Active 层 */

  /* 前景层级 */
  --fg-primary: #F1F5F9;        /* 主文本 */
  --fg-secondary: #94A3B8;      /* 次要文本 */
  --fg-tertiary: #475569;       /* 辅助/禁用文本 */
  --fg-accent: #38BDF8;         /* 强调色 — 天青蓝 */

  /* 语义色 — 产业链层级 */
  --tier-upstream: #34D399;     /* 上游 — 翡翠绿 */
  --tier-midstream: #60A5FA;    /* 中游 — 天蓝 */
  --tier-downstream: #FBBF24;   /* 下游 — 琥珀金 */
  --tier-general: #A78BFA;      /* 通用 — 紫罗兰 */

  /* 状态色 */
  --state-success: #22C55E;
  --state-warning: #F59E0B;
  --state-error: #EF4444;
  --state-info: #3B82F6;

  /* 边框与分割 */
  --border-subtle: rgba(255,255,255,0.06);
  --border-default: rgba(255,255,255,0.1);
  --border-strong: rgba(255,255,255,0.15);
}
```

**实施要点：**
- 所有组件统一使用 CSS 变量，禁止硬编码颜色
- 引入 `oklch()` 色彩空间保证 perceptual uniformity
- 增加 `data-theme="light"` 支持（未来扩展）

### 1.2 字体与排版系统

| `numeric` | 12px | 500 | 1 | Tabular-nums 数据 |

**字体栈：**
```css

```

**数字专用：**
```css

```

## 二、核心视觉层优化

### 2.1 节点设计重构（Node Redesign）

当前问题：固定 120×44px 矩形，信息展示单一，无数据密度，选中态微弱。

**新节点规格（3 种尺寸自适应）：**

```
┌─────────────────────────────────┐
│ ● 晶圆制造        [中]  ↑ 3     │  ← 顶部：名称 + 层级徽章 + 下钻指示
├─────────────────────────────────┤
│  半导体材料 → 晶圆加工 → 制造   │  ← 中间：一句话描述（hover 显示）
├─────────────────────────────────┤
│  📈 12只关联股票  │  📁 5子节点  │  ← 底部：数据指标
└─────────────────────────────────┘
```

**节点状态机设计：**

| 状态 | 视觉表现 | 动画 |
|------|---------|------|
| Default | `bg-surface` + `border-subtle` | — |
| Hover | 边框提亮 + 微上浮 `translateY(-2px)` + `shadow-md` | 200ms ease-out |
| Selected | 主色调边框 + `glow-selected` + 内部左侧 3px 色条 | 150ms ease |
| Active (按下) | `scale(0.98)` | 50ms ease |
| Dragging | `opacity: 0.9` + `shadow-lg` + 网格吸附预览 | — |
| Highlighted (搜索/联动) | 脉冲光环动画 | 1.5s ease infinite |
| Dimmed (非焦点) | `opacity: 0.35` + `filter: grayscale(0.6)` | 300ms ease |

**节点层级色彩编码（内嵌左侧色条）：**
```css
.node-tier-upstream::before { background: var(--tier-upstream); }
.node-tier-midstream::before { background: var(--tier-midstream); }
.node-tier-downstream::before { background: var(--tier-downstream); }
```

### 2.2 连线与流动效果（Edge Redesign）

当前问题：静态虚线 SVG path，无方向感，无数据流动语义。

**新连线系统：**

1. **贝塞尔曲线优化**：使用更平滑的 `cubic-bezier` 控制点，避免锐角
2. **方向指示器**：沿连线中点添加微动画箭头（SVG marker）
3. **数据流动动画**：
   - 默认：缓慢流动的光点（`stroke-dashoffset` 动画，2s 周期）
   - Hover 节点：相连边高亮 + 流动加速
   - 选中节点：流入边/流出边分色（绿=流入，蓝=流出）
4. **连线标签**：悬浮于连线上方，背景毛玻璃，避免与节点重叠

```tsx
// 连线动画伪代码
const animateFlow = (ctx, path, progress) => {
  const gradient = ctx.createLinearGradient(x1, y1, x2, y2);
  gradient.addColorStop(0, 'transparent');
  gradient.addColorStop(progress, 'rgba(56,189,248,0.6)');
  gradient.addColorStop(progress + 0.1, 'transparent');
  ctx.strokeStyle = gradient;
};
```

### 2.3 画布背景与环境

当前问题：纯色背景 `#0a0e1a`，缺乏空间感。

**新背景设计：**

1. **点阵网格**：极淡的点阵（`rgba(255,255,255,0.03)`），随缩放动态调整密度
2. ** subtle gradient**：从中心向四周极微弱的径向渐变，营造空间深度
3. **坐标指示器**：右下角显示当前视口的世界坐标（金融终端风格）

```css
.canvas-bg {
  background-color: var(--bg-base);
  background-image: 
    radial-gradient(circle at 50% 50%, rgba(56,189,248,0.03) 0%, transparent 70%),
    radial-gradient(circle, rgba(255,255,255,0.03) 1px, transparent 1px);
  background-size: 100% 100%, 24px 24px;
}
```

### 2.4 迷你地图（Minimap）

新增组件：右下角固定位置，显示整体图谱缩略图 + 当前视口矩形框，支持点击快速定位。

```tsx
<Minimap 
  nodes={allNodes}
  viewRect={{ x, y, width, height }}
  onNavigate={(worldX, worldY) => setPan({ x: -worldX * zoom, y: -worldY * zoom })}
/>
```

## 三、交互层优化（Interaction Design）

### 3.1 手势与导航系统

当前问题：缩放以按钮为主，平移需按住空白处，无惯性，体验生硬。

**新手势体系：**

| 输入 | 动作 | 行为 |
|------|------|------|
| 滚轮 | 画布缩放 | 以鼠标指针为中心缩放（非画布中心） |
| 滚轮 + Shift | 水平滚动 | 快速水平导航 |
| 中键拖拽 / 空格+拖拽 | 画布平移 | 惯性滑动（inertia: 衰减系数 0.9） |
| 双击节点 | 下钻 | 缩放动画过渡到下钻视图 |
| 双击空白 | 适应视图 | `fitView()` 动画 |
| 右键节点 | 上下文菜单 | 毛玻璃菜单，含快捷键提示 |
| 右键空白 | 添加节点菜单 | 快速创建 |
| 框选（Shift+拖拽） | 多选 | 矩形选框，批量操作 |
| 拖拽节点 | 移动位置 | 网格吸附（snap: 8px） |

**键盘快捷键映射：**

| 快捷键 | 功能 |
|--------|------|
| `Space` | 按住临时切换为平移模式 |
| `Ctrl/Cmd + F` | 聚焦搜索框 |
| `Ctrl/Cmd + 0` | 重置缩放 |
| `Ctrl/Cmd + +/-` | 放大/缩小 |
| `Escape` | 取消选择 / 关闭面板 |
| `Delete` | 删除选中节点（编辑模式） |
| `E` | 切换编辑模式 |
| `←↑↓→` | 方向键导航节点（按图谱拓扑） |
| `Enter` | 下钻到选中节点 |
| `Backspace` | 返回上一级 |
| `Tab` | 在节点间循环焦点 |

### 3.2 搜索与快速导航（Command Palette）

新增全局搜索组件（`Cmd+K` 唤起）：

```
┌─────────────────────────────────────────┐
│  🔍 搜索节点、股票、产业...              │
├─────────────────────────────────────────┤
│  📂 半导体产业                           │
│    → 晶圆制造                [中游]      │
│    → 光刻胶材料              [上游]      │

```

- 支持模糊搜索、拼音搜索（`jingyuan` → 晶圆）
- 结果按类型分组，键盘上下选择，Enter 跳转
- 使用 `cmdk` 库（已安装）快速实现

### 3.3 选中与焦点管理

**选中状态提升：**
- 选中节点：周围非关联节点 `dimmed`（淡化）
- 关联边高亮发光，非关联边降低透明度
- 右侧详情面板滑入动画（300ms cubic-bezier(0.16, 1, 0.3, 1)）

**焦点系统（Accessibility）：**
- 所有节点支持 `tabIndex`，键盘可达
- 焦点环使用 `ring-2 ring-accent ring-offset-2 ring-offset-bg-base`
- 屏幕阅读器播报节点名称、层级、关联数量

### 3.4 下钻过渡动画

当前问题：瞬间切换，无过渡，用户容易迷失。

**新过渡设计：**

```
1. 用户双击/Enter 下钻
2. 当前层级节点向外扩散淡出（150ms）
3. 连线收缩为点（100ms）
4. 新层级节点从中心放大进入（200ms，stagger 20ms）
5. 连线从新节点生长出来（150ms）
6. 面包屑更新动画（新段滑入）
```

使用 `FLIP` 动画技术实现平滑过渡。

### 3.5 编辑模式优化

当前问题：红色指示灯粗糙，操作入口不明显。

**新编辑模式设计：**

1. **状态指示**：顶部工具栏变为编辑主题色（蓝色边框 + 编辑图标旋转动画）
2. **拖拽手柄**：节点四角出现 6×6px 拖拽手柄，hover 显示
3. **连接创建**：从锚点拖拽出线，目标节点高亮可连接，释放创建连线
4. **撤销/重做**：`Ctrl+Z` / `Ctrl+Shift+Z` 支持操作历史栈
5. **批量选择**：框选后支持批量删除/移动/改色
6. **自动保存**：编辑操作 500ms 后自动 localStorage 保存，toast 提示

## 五、动画与动效系统

### 5.1 动效原则

- **目的性**：每个动画都服务于引导注意力或提供反馈
- **快速**：交互反馈 ≤ 150ms，过渡动画 ≤ 300ms
- **自然**：使用 `cubic-bezier(0.16, 1, 0.3, 1)`（ease-out-expo）为主要缓动
- **克制**：避免过度动画，金融工具以效率优先

### 5.2 动画清单

| 场景 | 时长 | 缓动 | 实现方式 |
|------|------|------|---------|
| 节点 Hover | 200ms | ease-out | CSS transition |
| 节点选中 | 150ms | ease-out-expo | CSS transition |
| 面板滑入 | 300ms | ease-out-expo | CSS transform |
| 下钻过渡 | 400ms | ease-in-out | FLIP + GSAP |
| 连线流动 | 2000ms | linear | SVG animation |
| 搜索打开 | 200ms | ease-out-expo | CSS transform + opacity |
| Toast 提示 | 300ms in, 5s hold, 300ms out | ease-out | CSS animation |
| 加载骨架 | 1.5s | ease-in-out | CSS shimmer |
| 数字变化 | 300ms | ease-out | CountUp animation |

### 5.3 性能保障

- 所有动画仅使用 `transform` 和 `opacity`（GPU 加速）
- 节点使用 `will-change: transform`（仅在动画时添加）
- 大量节点时使用 `content-visibility: auto`
- RAF 循环中避免 setState，使用 ref + imperative update

## 六、性能优化策略

### 6.1 渲染性能

1. **Canvas 渲染管线重构**：
   - 将 SVG 连线改为 Canvas 2D 渲染（单画布，无 DOM 开销）
   - 节点仍用 DOM（保证可访问性和交互），但使用 `transform3d`
   - 双画布架构：底层 Canvas 画连线和背景，上层 DOM 节点

2. **视口裁剪（Viewport Culling）**：
   ```tsx
   const visibleNodes = nodes.filter(n => 
     n.x + NODE_W > viewLeft && n.x < viewRight &&
     n.y + NODE_H > viewTop && n.y < viewBottom
   );
   ```

3. **连线 LOD（Level of Detail）**：
   - 缩放 < 0.5：连线简化为直线，隐藏标签
   - 缩放 < 0.2：隐藏连线，仅显示节点热力点

### 6.2 状态管理优化

当前问题：Home 组件状态集中，重渲染波及全树。

**重构方案：**
- 使用 React Context + useReducer 管理全局状态
- 节点交互状态（hover、drag）全部用 ref 管理，不触发 React 渲染
- 仅 `selectedNodeId`、`currentParentId` 等需要同步 UI 的状态走 React

### 6.3 数据加载优化

1. **骨架屏（Skeleton）**：产业切换时显示节点骨架占位
2. **渐进加载**：大数据量时先显示上游节点，再逐步加载中下游
3. **localStorage 缓存**：布局缓存 + 最近查看缓存

## 七、响应式与多端适配

### 7.1 断点设计

| 断点 | 宽度 | 布局调整 |
|------|------|---------|
| Desktop XL | ≥1600px | 全功能，侧边栏固定 |
| Desktop | 1280-1599px | 标准布局 |
| Tablet | 768-1279px | 侧边栏可收起，面板全宽覆盖 |
| Mobile | <768px | 单列布局，图谱全屏，面板底部抽屉 |

### 7.2 移动端适配

1. **手势**：双指缩放、单指平移、长按节点打开菜单
2. **面板**：底部抽屉式面板，可滑动关闭
3. **导航**：底部 Tab 栏替代顶部产业切换
4. **搜索**：全屏搜索页

## 八、可访问性（A11y）

### 8.1 键盘完全可达
- 所有交互元素可通过 Tab 到达
- 方向键在节点图中导航（使用 ARIA grid 或 tree 模式）
- Enter 选中/下钻，Escape 返回

### 8.2 屏幕阅读器支持
```tsx
<div role="tree" aria-label="产业链图谱">
  <div role="treeitem" aria-selected={isSelected} aria-expanded={hasChildren}>
    <span>{node.name}</span>
    <span className="sr-only">{tierLabel}, {stockCount}只关联股票, {childCount}个子节点</span>
  </div>
</div>
```

### 8.3 色彩无障碍
- 所有颜色组合满足 WCAG AA 对比度（≥4.5:1）
- 不仅依赖颜色传达信息（图标 + 文字 + 形状冗余）
- 支持 `prefers-reduced-motion` 媒体查询，关闭非必要动画
