# 跨产业一致性检查提示词

## 使用方式

1. 将 `app/src/data/` 目录下所有产业数据文件粘贴给 AI
2. AI 会跨产业比对规范一致性
3. 输出不一致清单 + 标准化建议 + 批量修复代码

---

## 提示词（复制使用）

```
请作为产业链数据规范审计员，检查所有产业数据文件的一致性。

## 输入数据

```typescript
// 产业1：半导体
【粘贴 semiconductor.ts 的完整代码】

// 产业2：电力
【粘贴 power.ts 的完整代码】

// 产业3：电网设备
【粘贴 gridEquipment.ts 的完整代码】

// ... 其他产业文件
```

## 检查维度

### 1. ID 分配规范

| 检查项 | 标准 |
|--------|------|
| 根节点 ID 段 | 各产业根节点 ID 范围不重叠 |
| 子节点 ID 段 | 各产业子节点 ID 范围不重叠 |
| 股票 ID 段 | 各产业股票 ID 范围不重叠 |
| 连接 ID 段 | 各产业连接 ID 范围不重叠 |
| ID 连续性 | 同产业内 ID 是否存在无意义的大间隙（合理预留除外） |

**输出格式：**
```
【ID规范】✅ 通过 / ❌ 问题
- 问题描述：XX产业的根节点ID 600 与 YY产业的根节点ID 600 冲突
- 建议：将 YY产业根节点调整为 700-799
```

### 2. 命名风格一致性

检查以下命名模式是否在各产业中统一：

| 维度 | 示例对比（不一致） | 建议统一 |
|------|-------------------|---------|
| 设备类命名 | "光刻设备" vs "刻蚀装备" | 统一为 "XX设备" |
| 材料类命名 | "正极材料" vs "硅钢材料" vs "绝缘料" | 统一为 "XX材料" 或 "XX料" |
| 路线类命名 | "磷酸铁锂" vs "TOPCon" vs "风电" | 技术路线统一为全称或通用简称 |
| 终端类命名 | "新能源车" vs "电站" vs "充电桩" | 保持多样性，但同一应用场景区分清晰 |

### 3. 描述深度一致性

检查各产业节点的 description 字段的信息密度：

| 等级 | 标准 | 示例 |
|------|------|------|
| A级（优秀） | 包含产业地位 + 量化数据（产能/市占/增速） | "全球出货第一，市占率32%，绑定宁德时代" |
| B级（合格） | 包含产业地位或功能描述 | "半导体制造核心环节，涉及光刻蚀刻等工艺" |
| C级（不足） | 仅一句话定性描述，无量化 | "芯片设计的基础工具" |

要求：同一产业内，根节点至少达到 B 级，子节点至少达到 A 级。

### 4. 连接标签风格

检查所有连接的 label 是否风格统一：

| 风格 | 示例 | 建议 |
|------|------|------|
| 动词短语 | "原材料供给"、"产品输送" | ✅ 推荐 |
| 名词短语 | "原材料"、"产品" | ❌ 避免，信息不足 |
| 箭头表达 | "A→B" | ❌ 避免，冗余 |

### 5. 股票标签规范

检查所有 stocks 中的 tag 字段：

| 规范 | 示例 |
|------|------|
| ✅ 包含核心竞争力 + 量化信息 | "光模块全球龙头，市占42%，1.6T已量产" |
| ❌ 太泛 | "龙头企业" |
| ❌ 仅分类 | "光模块公司" |
| ❌ 主观判断 | "值得关注" |

### 6. 层级划分一致性

检查同类节点在不同产业中的层级归属是否一致：

| 节点类型 | 半导体 | 电力 | 电网设备 | 一致性 |
|----------|--------|------|----------|--------|
| 原材料/矿产 | upstream | upstream | upstream | ✅ |
| 核心设备 | upstream | upstream | ? | 检查 |
| 制造/代工 | midstream | midstream | ? | 检查 |
| 终端产品 | downstream | downstream | ? | 检查 |

## 输出格式

```markdown
## 一致性检查报告

### 严重问题（需立即修复）
1. **ID冲突**：XX产业与YY产业的连接ID 60001 重复
   - 影响：运行时数据覆盖
   - 修复：将YY产业的连接ID改为70001起

### 风格不一致（建议统一）
1. **命名风格**：电力产业用"储能系统"，电网设备用"无功补偿/电容器"
   - 建议：统一为"XX系统"或"XX设备"，保持同层级命名规则一致

### 深度差异（逐步优化）
1. **描述深度**：半导体产业的子节点80%达到A级，电力产业仅50%
   - 建议：对电力产业子节点补充量化数据（市占率/产能/增速）

### 批量修复代码

```typescript
// 如需批量修复，输出可直接应用的代码块
// 例如：统一命名、补全描述等
```
```

## 执行指令

请按以下顺序执行：
1. 提取所有产业的节点、连接、股票列表
2. 逐维度检查一致性
3. 输出问题报告 + 批量修复建议
