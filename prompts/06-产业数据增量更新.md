# 产业数据增量更新提示词

## 使用方式

1. 将已有产业数据文件的完整代码 + 更新需求描述 粘贴给 AI
2. AI 会基于原数据，仅修改/新增指定部分，保留其他内容不变
3. 输出可直接替换原文件的完整代码

---

## 提示词（复制使用）

```
请作为产业链数据维护工程师，对已有产业数据进行增量更新。

## 约束（必须严格遵守）

1. **ID 不变**：已有节点的 ID 绝对禁止修改（会破坏 localStorage 位置缓存和连接关系）
2. **最小改动**：仅修改/新增用户明确要求的内容，未提及的节点、连接、描述保持原样
3. **ID 接续**：新增节点的 ID 必须从当前最大 ID + 1 开始，不跳号
4. **连接同步**：新增节点时，必须同步新增与其相关的连接；删除节点时，必须同步删除涉及该节点的所有连接
5. **数量一致**：修改后 childCount、stockCount 必须与实际数组长度一致

## 输入 1：当前数据文件（完整代码）

```typescript
【粘贴当前产业数据文件的完整代码】
```

## 输入 2：更新需求

【描述具体更新内容，例如】
- 新增：固态电池技术路线（作为中游技术路线C）
- 新增：3家固态电池概念股
- 修改：储能系统的描述（补充2024年装机数据）
- 删除：已退市的股票 XX公司
- 调整：将"XX节点"从上游移到中游

## 更新任务

### 第一步：理解更新范围

明确以下信息：
- 新增节点：N个（名称、层级、父节点、描述、大白话解释、股票）
- 修改节点：N个（具体字段变更）
- 删除节点：N个（ID列表）
- 新增连接：N条
- 删除连接：N条
- 修改描述：N处（专业描述或大白话解释）

### 第二步：分配新 ID

根据当前最大 ID 分配新 ID：

```
当前最大根节点ID: ___
当前最大子节点ID: ___
当前最大股票ID: ___
当前最大连接ID: ___

新增根节点ID: 当前最大 + 1, +2, ...
新增子节点ID: 当前最大 + 1, +2, ...
新增股票ID: 当前最大 + 1, +2, ...
新增连接ID: 当前最大 + 1, +2, ...
```

### 第三步：执行更新

#### 新增节点
- 在 rootNodes 中追加新的根节点（如新增根节点）
- 在 childNodes 中追加新的子节点数组（key 为父节点ID）
- **新增节点必须同时生成 plainLanguageDescription（大白话解释）**
- 更新父节点的 childCount

#### 修改节点
- 定位到具体节点，修改指定字段
- 如涉及层级变更，检查连接方向是否需要调整
- **如修改了 description，同步检查 plainLanguageDescription 是否仍准确，如不准确则同步更新**

#### 删除节点
- 从 rootNodes 或 childNodes 中删除
- 更新父节点的 childCount
- 删除该节点下的所有子节点（如有）
- 删除所有涉及该节点的连接

#### 连接更新
- 新增节点必须建立上下游连接
- 修改层级后，检查连接方向是否仍然正确

### 第四步：输出

输出**完整的更新后文件**（可直接替换原文件）：

```typescript
// === 更新后的完整文件 ===
[完整的 TypeScript 代码]
```

同时输出变更摘要：

```markdown
## 变更摘要

### 新增
- 根节点：N个（ID: ...）
- 子节点：N个（ID: ...）
- 股票：N只（ID: ...）
- 连接：N条（ID: ...）

### 修改
- 节点描述：N处（节点ID: ...）
- 大白话解释：N处（节点ID: ...）
- 节点层级：N处（节点ID: ...）
- 连接标签：N处

### 删除
- 节点：N个（ID: ...）
- 股票：N只（名称: ...）
- 连接：N条

### 自检结果
- [ ] 所有已有节点ID未变更
- [ ] 新增ID接续，无冲突
- [ ] childCount与实际长度一致
- [ ] stockCount与实际长度一致
- [ ] 无 dangling connection
- [ ] TypeScript语法正确
```

## 使用示例

**输入：**
```
【power.ts 完整代码】

更新需求：
1. 新增固态电池技术路线，作为储能系统(609)的子节点
2. 新增3只固态电池概念股：清陶能源、卫蓝新能源、赣锋锂业（固态电池业务）
3. 修改储能系统描述，补充"2024年中国固态电池产能规划超100GWh"
```

**AI 输出：**
```typescript
// 完整的更新后 power.ts 代码...
```

**变更摘要：**
- 新增子节点：1个（ID: 60032，固态电池，parentId: 609）
- 新增股票：3只（ID: 60093-60095）
- 修改描述：1处（储能系统节点）
- 更新 childCount：609的childCount从3改为4
```
