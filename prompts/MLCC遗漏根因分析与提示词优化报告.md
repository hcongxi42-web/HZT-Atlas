# MLCC 遗漏根因分析与提示词反向优化报告

## 一、问题现象

当前半导体产业链图谱包含 13 个根节点（EDA/IP核→分销渠道+光通信/CPO+被动元器件/MLCC），但**最初的提示词体系生成的半导体数据中完全没有 MLCC/被动元器件概念**。

MLCC（多层陶瓷电容器）是：
- 全球用量最大的被动元器件，手机单台用量 1000+ 颗
- A股相关上市公司 20+ 家（风华高科、三环集团、国瓷材料、顺络电子、火炬电子等）
- 2024年全球市场规模超 1000 亿元人民币

这样一个体量巨大的细分赛道，为什么最初的提示词完全没有生成？

---

## 二、根因分析（5层递进）

### 根因1：认知框架偏差 — 01-提示词的"大类产业"导向

**01-产业数据生成.md** 的核心框架：
```
上游至少4个一级概念，每个至少3个子概念
中游至少4个一级概念，技术路线独立分支
下游至少3个一级概念，包含终端+应用+服务/回收
```

这个框架是为**锂电池、光伏等独立产业**设计的。当 AI 用这套框架生成"半导体产业"时：
- 优先填充"芯片"核心环节（EDA、设备、材料、设计、制造、封测）
- MLCC 的上游（陶瓷粉体）只有2-3家核心公司，无法独立满足"至少3个子概念"的要求
- MLCC 的下游依附于终端应用，没有独立的"服务/回收"环节
- **AI 在框架约束下主动"裁剪"了 MLCC**

**本质**：提示词用"独立产业"的框架去套"产业内部细分节点"，导致细分节点被系统性遗漏。

---

### 根因2：知识库缺口 — 00-提示词缺少被动元器件映射

**00-产业层级确认.md** 的知识库：

| 用户可能说的名称 | 实际归属 | 能否独立 |
|-----------------|---------|---------|
| 碳化硅、氮化镓、HBM、光模块 | 半导体 | ❌ |
| 充电桩、虚拟电厂 | 电力/电网设备 | ❌ |
| ... | ... | ... |

**问题**：知识库只列出了"碳化硅、氮化镓、HBM、光模块"属于半导体细分节点，**完全没有提到 MLCC、PCB、连接器、晶振、继电器等配套元器件**。

AI 在没有外部知识的情况下，无法自行推断"被动元器件也属于半导体产业链"。这是一个典型的**"知识库不完整导致遗漏"**问题。

---

### 根因3：设计盲点 — 半导体初始结构聚焦"有源器件"（芯片）

现有半导体根节点列表：
- 上游：EDA/IP核、半导体设备、半导体材料、晶圆制造
- 中游：芯片设计、存储芯片、功率半导体、模拟芯片、逻辑芯片、光通信/CPO
- 下游：封装测试、终端应用、分销渠道

**所有这些节点都是围绕"芯片"（有源器件）的**。

提示词没有明确说明：
> "半导体产业链应包含芯片（有源器件）+ 被动元器件（MLCC/电阻/电感）+ PCB/连接器 等完整电子元器件"

AI 会默认 **"半导体 = 芯片"**，从而系统性地遗漏被动元器件。

**类比**：如果生成"汽车产业"只包含发动机、变速箱、底盘，却遗漏了轮胎、玻璃、座椅——虽然这些不是"核心动力部件"，但缺了它们车根本跑不了。

---

### 根因4：追溯悖论 — 材料级深度要求导致 MLCC 被排除

**01-提示词要求**：
> "每个分支必须追溯到最上游的原材料或核心装备，不能停在终端产品"

MLCC 的最上游是**钛酸钡等化工陶瓷粉体**，与半导体硅片、光刻胶等材料完全不同。

AI 在追溯时会发生以下推理：
1. MLCC 的上游是陶瓷粉体（化工材料）
2. 化工材料不属于"半导体材料"（硅片、光刻胶等）
3. 因此 MLCC 不属于半导体产业链

**错误！** MLCC 虽然不是用硅制造的，但它是**电子产品的必需元器件**，与芯片同处于一条产品价值链上。提示词的"材料级追溯"要求没有考虑到"跨材料体系的元器件配套"。

---

### 根因5：增量更新盲区 — 缺少"已有产业内部节点补充"规范

**01-提示词**只描述了如何生成"全新产业"的完整数据（3个代码块）。

**06-提示词**讲增量更新，但前提是用户**明确知道**要更新什么（"新增固态电池技术路线"）。

**问题**：如果 AI 在自检时发现了 MLCC 的缺失，它应该用什么格式输出补充内容？提示词没有给出规范。

现有提示词体系缺少一个关键能力：**让 AI 在生成/校验过程中主动发现产业内部遗漏，并按规范格式输出补充方案**。

---

## 三、反向优化方案

### 优化1：修改 00-产业层级确认.md — 补充知识库

在知识库中增加被动元器件等细分节点的映射：

```markdown
| 用户可能说的名称 | 实际归属 | 能否独立 |
|-----------------|---------|---------|
| 碳化硅、氮化镓、HBM、光模块 | 半导体 | ❌ |
| **MLCC、片式电感/电阻、晶振、连接器** | **半导体（配套元器件）** | **❌** |
| **PCB/覆铜板、印制电路板** | **半导体（配套元器件）** | **❌** |
| 充电桩、虚拟电厂 | 电力/电网设备 | ❌ |
```

同时增加"半导体配套元器件"分类说明：
> 半导体产业链不仅包含"芯片"（有源器件），还应包含**被动元器件**（MLCC、电阻、电感、晶振）、**PCB/连接器**、**封装基板**等配套元件。这些元件虽然技术门槛低于芯片，但在电子产品中用量巨大（手机中被动元器件数量是芯片的10倍以上），是产业链完整性不可忽视的部分。

---

### 优化2：修改 01-产业数据生成.md — 增加"配套元器件完整性"检查

在自检清单中增加以下检查项：

```markdown
#### 配套元器件完整性检查（电子产品 BOM 视角）

一个完整的电子产品 BOM（物料清单）不仅包含芯片，还包含大量配套元器件。生成产业数据时，请从 BOM 视角检查以下关键配套是否覆盖：

**半导体产业必须覆盖的配套元器件**：
- [ ] 被动元器件：MLCC（多层陶瓷电容）、片式电阻、片式电感、晶振
- [ ] PCB/连接器：覆铜板、PCB 制造、板对板/线对板连接器
- [ ] 封装材料：引线框架、封装基板、键合丝、塑封料
- [ ] 散热材料：导热界面材料、散热片、均热板

**判断标准**：
- 如果终端应用包含"手机/电脑/汽车/服务器"，则必须有配套元器件节点
- 配套元器件可以作为独立根节点（与光通信/CPO 同级），或作为子节点挂载在合适位置
- 配套元器件的上游材料（如陶瓷粉体、铜箔）可以与现有材料节点建立跨分支连接
```

---

### 优化3：修改 01-产业数据生成.md — 增加"已有产业内部节点补充"规范

在"输出格式"章节后增加：

```markdown
---

## 五、特殊情况：已有产业内部节点补充

当用户要求为**已有产业**补充一个新的细分节点时（如"半导体产业增加 MLCC"），请按以下规范输出：

### 与生成新产业的区别

| 对比项 | 生成新产业 | 已有产业内部补充 |
|--------|-----------|-----------------|
| 输出范围 | 完整的3个代码块（rootNodes/childNodes/connections） | 仅输出新增/修改的部分 |
| ID分配 | 全新ID段 | 接续该产业当前最大ID |
| 父节点 | 全部为 null（根节点） | 新增的根节点 parentId=null，子节点挂在新根下 |
| 已有数据 | 无 | 必须保留所有已有节点和连接不变 |

### 输出格式（已有产业内部补充）

**代码块1：新增 rootNodes**
```typescript
// 在 rootNodes 数组末尾追加
{ id: 新根节点ID, name: "节点名称", level: "层级", description: "...", plainLanguageDescription: "...", parentId: null, stockCount: 0, childCount: N, stocks: [] },
```

**代码块2：新增 childNodes**
```typescript
// 在 childNodes 对象中追加
新根节点ID: [
  { id: 子节点ID, name: "子概念", level: "层级", description: "...", plainLanguageDescription: "...", parentId: 新根节点ID, stockCount: N, childCount: 0, stocks: [
    { id: 股票ID, name: "公司", code: "XXXXXX.XX", tag: "核心竞争力+量化" },
  ]},
],
```

**代码块3：新增 connections + 修改父节点 childCount**
```typescript
// 新增连接
{ id: 连接ID, fromNodeId: XX, toNodeId: XX, label: "关系描述" },

// 父节点 childCount 更新说明
// 根节点 X 的 childCount 从 N 改为 N+1
```

### ID 接续规则

补充节点前，先查询该产业的当前最大 ID：
- 根节点最大 ID + 1
- 子节点最大 ID + 1
- 股票最大 ID + 1
- 连接最大 ID + 1
```

### 优化4：修改 02-数据校验审核.md — 增加"BOM 完整性"校验维度

在"节点逻辑校验"中增加：

```markdown
#### 新增校验维度：BOM 完整性

从电子产品物料清单（BOM）视角审查产业完整性：

**半导体产业 BOM 校验清单**：
- [ ] 有源器件：CPU/GPU/存储/模拟/功率/逻辑芯片（当前已覆盖）
- [ ] 被动元器件：MLCC、电阻、电感、晶振（当前缺失）
- [ ] 连接与载体：PCB、连接器、封装基板（当前缺失）
- [ ] 散热与结构：导热材料、散热片、外壳/结构件（当前缺失）

**校验逻辑**：
如果终端应用节点包含"手机/电脑/汽车/服务器"，但产业链中完全没有被动元器件、PCB 等配套节点，则判定为**"BOM 完整性缺失"**，标记为逻辑瑕疵。
```

## 四、优化后效果预期

1. **00-提示词**：AI 在判断"MLCC"时会正确归类为"半导体配套元器件细分节点"，不会误判为独立产业
2. **01-提示词**：AI 在生成半导体产业数据时，会主动检查 BOM 完整性，自动补充被动元器件节点
3. **01-提示词**：当用户要求"补充 MLCC"时，AI 能按规范格式输出增量代码，不需要输出整个文件
4. **02-提示词**：投研视角校验时会主动发现"BOM 完整性缺失"问题，提示补充被动元器件

## 五、举一反三：还可能遗漏哪些节点？

基于同样的根因分析，以下节点也可能被现有提示词体系遗漏：

| 遗漏节点 | 所属产业 | 遗漏原因 |
|---------|---------|---------|
| PCB/覆铜板 | 半导体 | 不属于"芯片"范畴，上游是铜箔/树脂 |
| 连接器/接插件 | 半导体 | 机械+电气 hybrid，不属于纯半导体 |
| 封装基板/引线框架 | 半导体 | 被封装测试节点笼统覆盖 |
| 导热材料/散热方案 | 半导体 | 材料体系不同（石墨/金属/相变材料）|
| 晶振/时钟器件 | 半导体 | 被归入"模拟芯片"但不够精确 |
| 继电器/接触器 | 电网设备 | 被归入"开关设备"但不够独立 |
| 电线电缆（低压）| 电网设备 | 被高压电缆笼统覆盖 |

**建议**：按本报告的优化方案修改提示词后，对三个产业数据文件进行一次"BOM 完整性"专项审查。
