# 数据修复执行提示词

## 使用方式

1. 将 `02-数据校验审核.md` 输出的问题报告 + 当前数据文件的完整代码 一起粘贴给 AI
2. AI 会基于问题列表，输出修复后的完整数据文件
3. 直接替换原文件，重新编译验证

---

## 提示词（复制使用）

```
你既是投研分析师，又是 TypeScript 代码工程师。请根据下方的问题报告，修复产业链数据文件中的错误。

## 输入 1：当前数据文件（完整代码）

```typescript
【粘贴当前产业数据文件的完整代码】
```

## 输入 2：校验发现的问题列表

【粘贴 02-数据校验审核 输出的问题报告】

## 修复任务

### 修复原则

1. **最小改动**：只修改报告中指出的问题，未涉及的节点/连接/描述保持原样
2. **连锁修复**：修复一个节点时，检查是否影响其他引用（如修改节点名称后，连接的 label 中若包含该名称也需同步）
3. **ID 不变**：已有节点的 ID 严禁修改（会破坏 localStorage 中的位置缓存和连接关系）
4. **数量一致**：修改 childCount、stockCount 后，确保与实际数组长度一致
5. **连接同步**：删除节点时，一并删除所有涉及该节点的连接；新增节点时，补全相关连接

### 修复步骤

1. 逐条阅读问题报告，按「严重错误 → 逻辑瑕疵 → 优化建议」的优先级处理
2. 对每个问题，在代码中定位具体位置
3. 执行修复，记录修改内容
4. 修复完成后，自检以下事项：
   - [ ] TypeScript 语法正确，可直接编译
   - [ ] 所有节点 ID 唯一，无重复
   - [ ] 所有连接的两端节点 ID 都在 rootNodes 或 childNodes 中存在
   - [ ] childCount 与实际 childNodes[key] 的长度一致
   - [ ] stockCount 与实际 stocks 数组长度一致
   - [ ] 无 dangling connection（指向已删除节点的连接）

### 输出格式

输出修复后的**完整数据文件**（可直接替换原文件）：

```typescript
// === 修复后的完整文件 ===
[完整的 TypeScript 代码]
```

同时输出修复摘要：

```markdown
## 修复摘要

### 已修复（N处）
1. **问题X** → 修改了...（文件位置/节点ID）
2. **问题Y** → 删除了...并同步删除了关联连接

### 未修复（如有）
- **问题Z**：原因（如需人工判断/数据缺失/超出当前文件范围）

### 新增/变更统计
- 新增节点：N个
- 删除节点：N个
- 修改描述：N处
- 新增连接：N条
- 删除连接：N条
- 修改连接：N条
```
```
