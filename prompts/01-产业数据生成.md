# 通用产业链图谱生成提示词（可直接同步项目）

## 使用方式

1. 将 `【产业名称】` 替换为目标产业（如：锂电池、光伏、机器人、创新药）
2. 将 `【产业ID范围】` 替换为分配的ID段（见下方ID分配表）
3. 将提示词交给AI，输出3个TypeScript代码块
4. 直接复制粘贴到项目 `src/data/【产业名】.ts` 文件中

## 一、产业链拆解框架（必须严格遵守）

### 拆解原则
1. **三阶架构**：上游（原材料/设备）→ 中游（核心制造/组件）→ 下游（终端/应用/服务）
2. **材料级深度**：每个分支必须追溯到最上游的原材料或核心装备，不能停在终端产品
3. **完整覆盖**：每个一级概念下至少3个子概念，每个子概念至少3只股票
4. **竞争维度**：存在多条技术路线时，必须作为独立子概念分别列出（如锂电池的磷酸铁锂 vs 三元）

### 上游必须包含的分支（至少4个一级概念）
- **矿产/原材料**：最基础的矿产资源或化工原料（如锂矿、钴镍、磷化工、萤石）
- **核心材料**：经过加工的功能性材料（如正极材料、负极材料、电解液、隔膜、硅片）
- **核心设备**：生产制造所需的关键装备（如锂电设备、光伏设备、光刻机、沉积设备）
- **零部件/组件**：设备或产品所需的精密零部件（如轴承、阀门、泵、传感器、光学元件）

### 中游必须包含的分支（至少4个一级概念）
- **核心制造/主材加工**：产业链价值最高的制造环节（如电芯制造、晶圆代工、组件封装）
- **关键组件/模块**：组装所需的模块化部件（如BMS电池管理系统、逆变器、减速器模组）
- **技术路线A**：主流技术路径1（如磷酸铁锂电池、PERC电池、谐波减速器）
- **技术路线B**：主流技术路径2（如三元锂电池、TOPCon电池、RV减速器）
- **（可选）技术路线C**：新兴技术路径（如固态电池、钙钛矿、人形机器人关节）

### 下游必须包含的分支（至少3个一级概念）
- **终端产品**：直接面向消费者的成品（如新能源汽车、手机、电站、工业机器人）
- **应用场景**：B端行业应用（如储能系统、数据中心、汽车产线、医疗手术）
- **服务/回收**：后生命周期服务（如电池回收、电站运维、设备检修、梯次利用）

## 二、每个子概念的研究深度标准（对标专业研报）

每个子节点需要同时输出 **专业描述**（给投研人员）和 **大白话解释**（给完全外行的人）：

| 字段 | 目标读者 | 写作要求 | 示例（晶圆制造） |
|------|---------|---------|-----------------|
| `description` | 投研人员 | 专业、量化、有深度 | "半导体制造核心环节，涉及硅片加工、光刻、蚀刻等工艺，台积电/中芯国际为核心厂商" |
| `plainLanguageDescription` | 完全外行 | 不用术语、生活比喻、说人话 | "把沙子（硅）做成超级精密的电路板，相当于芯片的'印刷厂'。没有它就没有手机电脑里的芯片。" |

### 大白话解释写作规范（必须严格遵守）

1. **不用专业缩写**：EDA → "芯片设计软件"；IP核 → "芯片里的功能模块设计图"
2. **用生活比喻**：光刻机 → "芯片制造的照相机，把电路图'拍'到硅片上"
3. **说明价值**：不是"这是什么"，而是"这有什么用、为什么重要"
4. **关联熟悉事物**：IGBT → "电动车里控制电流的'开关'，决定了车能跑多快、多省电"
5. **一句话长度**：控制在 30-60 字，节点卡片上能直接显示
6. **严禁行为**：
   - ❌ 复制粘贴 description 的内容
   - ❌ 使用"某某领域的重要环节"这类空话
   - ❌ 出现英文缩写未解释

每个子节点的描述和股票标签必须包含以下至少2个维度的信息：

| 维度 | 示例（CPO光模块） | 示例（锂电池正极） |
|------|------------------|-------------------|
| 技术参数 | "800G/1.6T速率，月产能20万只" | "能量密度180-200Wh/kg，循环寿命3000次+" |
| 竞争地位 | "全球市占率42%，CR3超60%" | "全球出货量第一，市占率28%" |
| 国产替代 | "国内唯一量产，良率70-75%" | "国产化率85%+，海外客户认证中" |
| 供需缺口 | "供需缺口70%，价格涨180%" | "高端产品供不应求，溢价15%" |
| 客户绑定 | "英伟达核心供应商，GB300标配" | "宁德时代第一大供应商，占比40%" |
| 技术路线 | "InP EML为主，TFLN小批量" | "磷酸铁锂为主，高镍三元放量中" |

## 三、股票标签规范

标签必须是一句话，包含**核心竞争力+量化信息**：

**好标签**：
- "光模块全球龙头，市占42%，1.6T已量产" 
- "InP衬底国内唯一6英寸量产，良率70-75%"
- "磷酸铁锂正极出货全球第一，绑定宁德时代"
- "碳酸锂自有矿占比60%，成本优势显著"

**差标签**（避免）：
- "龙头企业"（太泛，无量化）
- "光模块公司"（只是分类，无竞争力描述）
- "值得关注"（主观判断，无事实支撑）

### 代码块3：connections（连接关系，追加到文件）

**连接分两层：根节点骨架连接 + 子节点精细连接。**

#### 第一层：根节点骨架连接（必须）

在 `rootNodes` 的节点之间建立**汇总级流向连接**，确保根视图下能看到产业层级的骨架线条。这些连接的 `fromNodeId` 和 `toNodeId` 必须是根节点 ID。

```typescript
export const connections: LocalConnection[] = [
  // ... 已有连接 ...
  
  // === 根节点骨架连接（根视图可见）===
  { id: 根连接ID1, fromNodeId: 上游根ID, toNodeId: 中游根ID, label: "原材料供给" },
  { id: 根连接ID2, fromNodeId: 中游根ID_A, toNodeId: 中游根ID_B, label: "加工转化" },
  { id: 根连接ID3, fromNodeId: 中游根ID, toNodeId: 下游根ID, label: "产品供应" },
  
  // === 【产业名称】子节点精细连接 ===
  // 上游→中游
  { id: 连接ID1, fromNodeId: 上游子ID, toNodeId: 中游子ID, label: "原材料供给" },
  // 中游→中游（技术路线间）
  { id: 连接ID2, fromNodeId: 技术A子ID, toNodeId: 技术B子ID, label: "技术替代/互补" },
  // 中游→下游
  { id: 连接ID3, fromNodeId: 中游子ID, toNodeId: 下游子ID, label: "产品供给" },
  // 下游→服务（回收）
  { id: 连接ID4, fromNodeId: 终端ID, toNodeId: 回收ID, label: "梯次利用/回收" },
];
```

**根节点骨架连接规则：**
1. 上游根节点必须至少有一条出边指向中游根节点
2. 中游根节点之间若存在加工依赖，必须建立连接
3. 中游根节点必须至少有一条出边指向下游根节点
4. 下游根节点可以有入边，但**不应有下游→下游的连接**（并列终端应用间无供需关系）
5. 骨架连接标签使用概括性动词短语（如"原材料供给"、"加工转化"、"产品供应"）

## 五、研究深度检查清单（AI自检）

生成完成后，请逐项确认：

- [ ] 上游至少有4个一级概念，每个至少3个子概念
- [ ] 中游至少有4个一级概念，技术路线作为独立分支列出
- [ ] 下游至少有3个一级概念，包含终端+应用+服务/回收
- [ ] 每个子概念至少有3只股票，最多不超过10只
- [ ] 所有股票标签包含量化信息（市占率/产能/良率/增速等）
- [ ] 所有描述追溯到最上游原材料或核心装备
- [ ] 存在多条技术路线时，有独立的子概念和对比
- [ ] 连接关系覆盖了上→中、中→中、中→下、下→回收
- [ ] **根节点骨架连接**：rootNodes 之间至少有一条汇总级流向连接，确保根视图可见线条
- [ ] 所有ID在分配的范围内，不与其他产业冲突
- [ ] 股票代码格式正确（6位数字+.SZ/.SH/.BJ）

#### 配套元器件完整性检查（电子产品 BOM 视角）

一个完整的电子产品 BOM（物料清单）不仅包含芯片，还包含大量配套元器件。生成产业数据时，请从 BOM 视角检查以下关键配套是否覆盖：

**半导体产业必须覆盖的配套元器件**：
- [ ] 被动元器件：MLCC（多层陶瓷电容）、片式电阻、片式电感、晶振
- [ ] PCB/连接器：覆铜板、PCB 制造、板对板/线对板连接器
- [ ] 封装材料：引线框架、封装基板、键合丝、塑封料
- [ ] 散热材料：导热界面材料、散热片、均热板

**判断标准**：
- 如果终端应用包含"手机/电脑/汽车/服务器"，则必须有配套元器件节点
- 配套元器件可以作为独立根节点（与光通信/CPO 同级），或作为子节点挂载在合适位置
- 配套元器件的上游材料（如陶瓷粉体、铜箔）可以与现有材料节点建立跨分支连接

## 五、特殊情况：已有产业内部节点补充

当用户要求为**已有产业**补充一个新的细分节点时（如"半导体产业增加 MLCC"），请按以下规范输出：

### 与生成新产业的区别

| 对比项 | 生成新产业 | 已有产业内部补充 |
|--------|-----------|-----------------|
| 输出范围 | 完整的3个代码块（rootNodes/childNodes/connections） | 仅输出新增/修改的部分 |
| ID分配 | 全新ID段 | 接续该产业当前最大ID |
| 父节点 | 全部为 null（根节点） | 新增的根节点 parentId=null，子节点挂在新根下 |
| 已有数据 | 无 | 必须保留所有已有节点和连接不变 |
