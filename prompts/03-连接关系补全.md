# 连接关系补全提示词

## 使用方式

1. 将当前产业数据文件（`semiconductor.ts` / `power.ts` / `gridEquipment.ts` 等）的完整代码粘贴给 AI
2. AI 会基于已有节点，系统性地补全遗漏的连接关系
3. 将输出的 connections 代码块追加到原文件的 connections 数组中

---

## 提示词（复制使用）

```
请基于以下已有的产业链节点，系统性地补全所有合理的连接关系。

## 输入数据

【粘贴当前产业数据文件的完整代码】

## 补全任务

### 第一步：列出所有节点

先从代码中提取所有节点（rootNodes + childNodes 中的所有子节点），列出：
- 节点ID
- 节点名称
- 所属层级（upstream/midstream/downstream/general）
- 父节点ID（如有）

### 第二步：逐对检查连接关系

对每一对节点（A → B），按以下 checklist 判断是否应该建立连接：

**必须检查的连接类型：**

| 类型 | 判断标准 | 示例 |
|------|---------|------|
| 上游→中游 | A的产出物是B的直接原材料或核心设备 | 正极材料 → 电芯制造 |
| 中游→中游 | A和B存在技术互补、工艺协同或模块集成关系 | 光芯片 → 光模块 |
| 中游→下游 | A的成品是B的核心组件或动力系统 | 电芯 → 新能源车 |
| 下游→下游 | A和B存在应用场景的协同或客户重叠 | 充电桩 → 虚拟电厂 |
| 下游→服务 | A的产品生命周期结束后由B处理 | 退役电池 → 电池回收 |
| 同层级反馈 | B的需求变化会反向影响A的产能/价格 | 电站装机量 → 硅片需求 |

**禁止建立的连接：**
- 没有直接物料/价值/信息流转的间接关系（如"锂矿 → 新能源车"，应拆分为锂矿→正极→电芯→新能源车）
- 跨度过大的跳级连接（除非该跳级是产业现实中的直接交易关系）
- 同一节点自环（A → A）

### 第三步：输出格式

仅输出新增的连接代码块，格式如下：

```typescript
// === 补全的连接关系 ===
{ id: 连接ID, fromNodeId: 起点ID, toNodeId: 终点ID, label: "关系描述" },
// ...
```

**规则：**
1. 连接ID必须接续当前最大连接ID + 1，不冲突
2. 只输出**新增**的连接，已有连接不要重复输出
3. 每条连接必须有 label，描述价值流转方向
4. 新增连接数量不少于 5 条（如果确实没有遗漏，则输出"经检查，当前连接已完整"）

### 第四步：自检

输出完成后，请自检：
- [ ] 新增连接的 fromNodeId 和 toNodeId 都在已有节点列表中存在
- [ ] 没有与已有 connections 重复的连接（相同 fromNodeId + toNodeId 组合）
- [ ] label 描述的是价值/物料/信息的流动方向
- [ ] 连接ID在当前产业的ID范围内，不与其他产业冲突
```
